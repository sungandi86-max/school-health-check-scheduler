# Security Policy

## 목적

School Health Hub는
학교에서 사용하는 업무 시스템입니다.

학생 개인정보를 최소한으로 사용하고
공용 화면에서는 개인정보가 노출되지 않도록 설계합니다.

---

# 보호 대상

다음 정보를 보호합니다.

- 학생 이름
- 학번
- 학급
- 검사 결과
- 세션 정보

---

# 개인정보 원칙

학생 개인정보는

운영에 필요한 화면에서만 사용합니다.

공용 화면에는

학생 이름

학번

을 기본 표시하지 않습니다.

---

# 공용 화면 정책

다음 화면은 개인정보를 표시하지 않습니다.

- Display
- 관리자 현황판
- 교사용 현황판(기본)

공용 화면은

학급

진행률

통계

중심으로 제공합니다.

---

# 저장 정책

지원 저장 방식

- localStorage
- Supabase

localStorage

개인 개발

단독 테스트

학교 내부 사용

Supabase

여러 기기 공유

실시간 동기화

운영용

---

# Supabase 정책

- Row Level Security(RLS) 사용
- service_role Key는 프론트엔드에서 사용하지 않음
- anon Key만 클라이언트 사용
- 정책 없이 테이블 공개 금지

---

# 브라우저 보안

다음 사항을 유지합니다.

- HTTPS
- 환경변수 사용
- API Key 하드코딩 금지
- Console에 개인정보 출력 금지
- URL에 학생정보 포함 금지

---

# 현장 모드 정책

현장 모드는

운영에 필요한 최소 정보만 표시합니다.

불필요한 개인정보는 표시하지 않습니다.

---

# 현황판 정책

현황판은

학생이 아니라

학급

진행률

운영 상태

를 중심으로 제공합니다.

---

# 개발 원칙

새 기능을 추가할 때

반드시 다음을 확인합니다.

□ 개인정보 노출 여부

□ 공용 화면 노출 여부

□ 저장 위치

□ 접근 권한

□ URL 노출

□ Console 출력
