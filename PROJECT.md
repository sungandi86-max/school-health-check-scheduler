# School Health Hub

## 현재 버전
v1.0 MVP

## 프로젝트 목적
School Health Hub는 보건교사용 학교 보건 업무 허브입니다.
현재 이 저장소에서는 학교에서 직접 운영하는 별도검사 업무를 준비, 운영, 현황 공유, 결과 정리까지 지원하는 별도검사 Workspace를 개발합니다.

## 제품 구조

School Health Hub
- 별도검사 Workspace
- 온라인 보건실
- AI 업무도우미
- 향후 보건교사용 업무 도구

## 별도검사 Workspace 구조

별도검사 Workspace는 결핵검진과 소변검사를 중심으로 운영합니다.

기본 흐름:

운영 대시보드
→ 학교 설정
→ 검사 조건
→ 대상 학급
→ 시간표 생성
→ 현장 모드
→ 현황판
→ 운영 보고서

## 주요 개념

### 운영 대시보드
검진별 최근 작업, 세션 선택, 이어하기, 운영 시작, 현황판 진입을 담당하는 시작 화면입니다.

### 운영 준비
학교 설정, 검사 조건, 대상 학급, 학급별 검진 순서, 분반·혼합수업 참고자료를 정리하는 단계입니다.

### 현장 모드
검진 현장에서 명렬표를 보며 학생별 상태를 체크하는 화면입니다.
특정 사용자용이 아니라 현장에 있는 사람이 사용하는 모드입니다.
결핵검진은 검진 완료 여부를, 소변검사는 검체 제출 여부를 확인하는 데 사용합니다.

### 현황판
보건교사, 교사, 관리자, Display 화면에서 진행 상황을 확인하는 화면입니다.
공용 화면에는 학생 이름과 학번을 기본 표시하지 않습니다.

### 운영 보고서
검진 종료 후 미검 학생, 운영 로그, 진행 결과를 확인하고 출력하거나 보관하는 화면입니다.

## 저장 구조

현재 MVP는 localStorage 기반으로도 동작합니다.
Supabase는 여러 기기 실시간 공유가 필요할 때 사용하는 선택 저장 방식입니다.

- localStorage: 단독 테스트, 개발, 개인정보 외부 저장 최소화
- Supabase: 보건교사 PC, 현장 태블릿, Display, 교사용/관리자 화면 동기화용

## 개인정보 원칙

- 학생 개인정보는 운영에 필요한 화면에서만 사용합니다.
- 교사용 현황판, 관리자 현황판, Display 화면에는 학생 이름과 학번을 기본 표시하지 않습니다.
- 공용 화면은 학급 단위 진행 정보와 통계 중심으로 표시합니다.

## 개발 원칙

- 새 프로젝트를 만들지 않고 기존 구조 안에서 수정합니다.
- 기능 삭제나 큰 구조 변경은 명시 요청이 있을 때만 합니다.
- Repository, DataService, StorageAdapter, Supabase 로직은 필요한 경우에만 최소 수정합니다.
- 계산 로직은 별도 요청 없이는 수정하지 않습니다.
- PC는 설정·생성·보고서 중심, 모바일은 확인·운영 모니터링 중심, 태블릿은 현장 모드 중심으로 설계합니다.
- 화면 문구는 보건교사가 실제 업무에서 이해하기 쉬운 표현을 우선합니다.
