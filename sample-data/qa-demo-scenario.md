# QA 데모 시나리오 상태 예시

이 문서는 `qa-demo-roster.csv`를 업로드한 뒤 검진 당일 상황을 빠르게 재현하기 위한 안내입니다.

## 기본 세션

| 항목 | 예시 |
| --- | --- |
| 세션명 | 2027 결핵검진 QA |
| 검사일 | 2027-06-24 |
| 장소 | 보건실 앞 검진장 |
| 대상 학년 | 2, 3 |
| 검사 순서 | 2-1부터 2-12, 3-1부터 3-12 |

## 상태 입력 예시

| 구분 | 입력 예시 | 확인 화면 |
| --- | --- | --- |
| 현재 학급 | 2-3 | 운영센터, 교사용 현황판, 관리자 현황판, Display |
| 다음 학급 | 2-4 | 운영센터, 교사용 현황판, Display |
| 다다음 학급 | 2-5 | Display Portrait, 관리자 현황판 |
| 완료 학급 | 2-1, 2-2 | 관리자 현황판, 운영 보고서 |
| 미도착 학급 | 2-4 | 운영센터, 관리자 현황판, Display Landscape |
| 지연 상황 | 지연 시간 7분 | 운영센터, 관리자 현황판, Display Landscape |
| 안내 문구 | 가방은 교실에 두고 이동하세요. | 교사용 현황판, Display |

## 학생 상태 예시

| 학급 | 학생 코드 | 상태 | 목적 |
| --- | --- | --- | --- |
| 2-1 | QA학생-2-01-01 | 완료 | 완료 학생 표시 확인 |
| 2-1 | QA학생-2-01-02 | 완료 | 완료율 증가 확인 |
| 2-2 | QA학생-2-02-01 | 결석 | 결석 학생 구분 확인 |
| 2-3 | QA학생-2-03-01 | 완료 | 현재 학급 일부 완료 확인 |
| 2-3 | QA학생-2-03-02 | 미완료 | 미완료 학생 수 확인 |
| 2-4 | QA학생-2-04-01 | 미완료 | 미도착 학급과 미완료 구분 확인 |
| 3-1 | QA학생-3-01-01 | 완료 | 3학년 데이터 표시 확인 |

## 화면별 확인 포인트

| 화면 | 확인 포인트 |
| --- | --- |
| 운영센터 | 현재/다음/미도착/지연 상태가 한 화면에서 구분된다 |
| Tablet Mode | 완료, 미도착, 이전, 다음 버튼이 크게 보인다 |
| 교사용 현황판 | 학생 이름과 번호 없이 학급 이동 정보만 표시된다 |
| 관리자 현황판 | 완료 학급, 미도착 학급, 지연 시간, 최근 로그가 보인다 |
| Display Portrait | 현재 학급이 가장 크게 보이고 안내 문구가 읽힌다 |
| Display Landscape | 현재/다음/완료/남은 학급과 진행률이 보인다 |
| 운영 보고서 | 요약 카드와 학급별 결과가 인쇄 형태로 정리된다 |

## 예외 상황 점검

- 결석 학생은 완료 학생과 다른 상태로 표시되는지 확인합니다.
- 미도착 학급은 완료 학급과 구분되는 색상/상태로 표시되는지 확인합니다.
- 지연 시간을 입력하면 관리자 현황판과 Display Landscape에 반영되는지 확인합니다.
- Supabase 환경변수가 없는 상태에서도 앱이 localStorage 모드로 동작하는지 확인합니다.
- 모바일/태블릿 화면에서 가로 스크롤이나 버튼 잘림이 없는지 확인합니다.
