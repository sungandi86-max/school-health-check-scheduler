# NEXT

학교 별도검사 운영 도우미 v1.0 MVP 이후에 확인하고 정리할 작업 목록입니다.

## v1.0 이후 우선 작업

- Supabase 실전 동기화 테스트
  - 운영센터, 교사용 현황판, 관리자 현황판, Display Mode가 여러 기기에서 같은 세션을 안정적으로 공유하는지 확인합니다.
  - Supabase 미설정 또는 연결 실패 시 localStorage fallback이 계속 동작하는지 점검합니다.

- 학교별 저장소 운영 방식 검토
  - 단일 PC localStorage 운영, 보건실 전용 기기 운영, Supabase 공유 운영 중 학교 환경에 맞는 권장 방식을 정리합니다.
  - 개인정보 처리 기준과 교직원 내부 공유 범위를 학교별로 확인합니다.

- 실제 검진 시나리오 QA
  - Home에서 세션 선택 후 학교 설정, 검사 조건, 시간표 생성, 실시간 운영, 보고서까지 이어지는 전체 흐름을 실제 운영 순서로 점검합니다.
  - 결석 학생, 미도착 학급, 지연, 새로고침, 모바일 화면, 인쇄 화면을 함께 확인합니다.

- 보건실 세로형 Display 실기기 테스트
  - 보건실 앞 모니터, 삼성 MagicInfo 등 세로형 사이니지에서 현재/다음/다다음 학급과 안내 문구가 멀리서도 읽히는지 확인합니다.
  - 학생 개인정보가 표시되지 않는지 재확인합니다.

- 태블릿 현장 테스트
  - 검진장 검사자가 Tablet Mode에서 완료, 미도착, 이전, 다음 버튼을 한 손으로 누르기 쉬운지 확인합니다.
  - 태블릿 세로/가로 화면에서 핵심 카드와 버튼이 잘리지 않는지 점검합니다.

- 관리자 제안서/운영 매뉴얼 정리
  - 학교 관리자에게 설명할 제품 개요, 개인정보 보호 원칙, 운영 흐름, 도입 전 확인 사항을 정리합니다.
  - 보건교사용 현장 운영 매뉴얼과 검진 당일 체크리스트를 문서화합니다.

## 릴리스 이후 메모

- v1.0 MVP에서는 기능 추가보다 실제 검진 현장 검증과 문서 정리를 우선합니다.
- Repository, DataService, StorageAdapter, 계산 로직 변경이 필요한 작업은 별도 이슈로 분리합니다.
