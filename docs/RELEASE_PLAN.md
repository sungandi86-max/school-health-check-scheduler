# School Health Hub 릴리스 계획

이 문서는 `school-health-check-scheduler` 저장소를 School Health Hub 제품으로 확장하기 위한 GitHub 릴리스 버전 체계를 정리합니다.

## v1.0.0: 기존 학생건강검진 스케줄러

### 목표
기존 소변검사 및 결핵검진 시간표 자동배정 기능을 안정적인 기준 버전으로 정의합니다.

### 주요 기능
- 소변검사 방문형 시간표 자동배정
- 결핵검진 이동형 시간표 자동배정
- 학교 일과표, 검사 가능 시간, 수업 시간표 기반 배정
- CSV 다운로드, 인쇄, 안내 문구 복사
- 브라우저 localStorage 기반 저장

### 완료 기준
- 기존 스케줄러 핵심 기능 유지
- `npm run build` 통과
- 기존 사용 흐름에서 주요 회귀 없음

### 비고
- School Health Hub로 확장하기 전의 기준선 릴리스입니다.

## v2.0.0-alpha: 공통 건강검진 구조 리팩터링

### 목표
검사 종류별 분기 구조를 공통 HealthCheck 개념으로 정리합니다.

### 주요 기능
- `HealthCheckType` 기반 검사 종류 정의
- 소변검사, 결핵검진, 일반 건강검진, 기타검진 타입 기반 구조
- 검사 종류별 저장 데이터 분리 기반 마련
- 운영센터와 스케줄러가 공통 타입을 공유하도록 정리

### 완료 기준
- 기존 소변검사/결핵검진 스케줄 생성 유지
- 공통 타입과 유틸 추가
- `npm run build` 통과

### 비고
- alpha 단계에서는 내부 스케줄러 엔진의 기존 이름과 로직을 최대한 유지합니다.

## v2.1.0-alpha: 명렬표 관리

### 목표
검사 종류와 무관하게 사용할 수 있는 학생 명렬표 업로드 및 상태 관리 구조를 구축합니다.

### 주요 기능
- 엑셀 명렬표 업로드
- 학년, 반, 번호, 이름 열 자동 인식
- 검사 종류별 학생 목록 localStorage 저장
- 학생별 상태 관리: 대기, 완료, 결석, 조퇴, 지각, 추후검진
- 미검 학생 자동 집계

### 완료 기준
- 명렬표 업로드 후 학급별 학생 목록 표시
- 학생 상태 변경 및 메모 입력 가능
- 검사 종류별 데이터 분리 저장
- `npm run build` 통과

### 비고
- 서버와 DB 없이 localStorage 기반으로 운영합니다.

## v2.2.0-beta: 검진 운영센터

### 목표
보건교사가 검진 당일 진행 상황을 한 화면에서 운영할 수 있는 운영센터를 구축합니다.

### 주요 기능
- 현재 검사반, 다음 검사반, 예상 종료시간 표시
- 진행중/완료/미도착/지연 학급 현황
- 명렬표 기반 학생 체크 UI
- 운영 메모 작성

### 완료 기준
- 운영센터 화면에서 진행 상태와 학생 체크 흐름이 연결됨
- 기존 스케줄러 화면과 자연스럽게 공존
- `npm run build` 통과

### 비고
- beta 단계부터 실제 운영 시나리오 중심의 UI 검증을 시작합니다.

## v2.3.0-beta: 교사용 대시보드

### 목표
교사가 실시간으로 현재 검진 진행 상황을 확인할 수 있는 대시보드를 제공합니다.

### 주요 기능
- 현재 검사반 및 다음 검사반 표시
- 미도착 학급 및 지연 현황 표시
- 교사용 안내 문구 제공
- 모바일/PC 확인 화면 구성

### 완료 기준
- 교사용 대시보드에서 핵심 운영 상태 확인 가능
- 보건교사용 화면과 동일한 상태 기준 사용
- `npm run build` 통과

### 비고
- 실시간 동기화 전까지는 같은 브라우저 저장 상태 기반으로 검증합니다.

## v2.4.0-beta: 실시간 동기화

### 목표
여러 기기에서 검진 진행 상태를 동시에 확인할 수 있도록 동기화 구조를 도입합니다.

### 주요 기능
- 실시간 상태 공유를 위한 데이터 모델 확정
- Supabase 또는 Firebase 연동 검토
- 운영 상태, 학생 상태, 로그 동기화
- 네트워크 오류 및 재시도 처리 설계

### 완료 기준
- 최소 2개 이상의 기기에서 상태 동기화 확인
- localStorage 기반 흐름과의 마이그레이션 경로 정리
- `npm run build` 통과

### 비고
- 실제 학교 현장 네트워크 환경에서 별도 검증이 필요합니다.

## v2.5.0-beta: 관리자 대시보드

### 목표
검진 운영 전체 현황을 관리자가 파악할 수 있는 대시보드를 제공합니다.

### 주요 기능
- 검사 종류별 운영 현황
- 학년/학급별 완료율
- 미검 학생 요약
- 지연 및 특이사항 현황

### 완료 기준
- 관리자 관점에서 전체 운영 상태 확인 가능
- 운영센터, 교사용 대시보드와 동일한 데이터 구조 사용
- `npm run build` 통과

### 비고
- 권한 관리가 필요한 경우 인증 구조와 함께 설계합니다.

## v2.6.0-beta: 운영 로그 및 보고서

### 목표
검진 당일 운영 기록을 보고서와 사후 확인 자료로 남길 수 있게 합니다.

### 주요 기능

- 학생 상태 변경 이력
- 미검/추후검진 대상자 보고서
- CSV 또는 문서 형식 내보내기

### 완료 기준
- 운영 종료 후 핵심 보고 자료 생성 가능
- 개인정보 노출 최소화 원칙 반영
- `npm run build` 통과

### 비고
- 보고서 양식은 학교 현장 피드백을 반영해 조정합니다.

## v3.0.0: School Health Hub 첫 정식 버전

### 목표
스케줄러, 명렬표, 운영센터, 대시보드, 보고서를 통합한 School Health Hub 정식 버전을 릴리스합니다.

### 주요 기능
- 공통 건강검진 모듈
- 검사 종류별 스케줄 생성
- 학생 명렬표 및 상태 관리
- 검진 운영센터
- 교사용/관리자 대시보드
- 실시간 동기화
- 운영 로그 및 보고서

### 완료 기준
- 주요 기능 통합 검증 완료
- 문서, 릴리스 노트, 배포 절차 정리
- 개인정보 보호 및 현장 운영 체크리스트 완료
- `npm run build` 통과

### 비고
- v3.0.0부터 제품명은 School Health Hub를 공식 사용합니다.
