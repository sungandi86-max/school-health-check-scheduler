# 실사용 QA 체크리스트

학교 별도검사 운영 도우미를 실제 검진 당일 흐름으로 점검하기 위한 수동 QA 문서입니다.

이 문서는 기능 개발용 테스트가 아니라, 보건교사 PC, 태블릿, 보건실 Display, 교사용/관리자 현황판이 실제 운영 순서대로 문제없이 이어지는지 확인하는 데 사용합니다.

## 테스트 전 준비

- 실제 학생 이름, 학번, 건강 정보는 사용하지 않습니다.
- 테스트용 세션명은 예: `2027 결핵검진 QA`, `2027 소변검사 QA`처럼 구분합니다.
- 브라우저를 새로고침해도 데이터가 유지되는지 확인하기 위해 같은 브라우저에서 한 번 이상 새로고침합니다.
- Supabase 모드 점검 시에는 PC, 태블릿, Display 화면을 서로 다른 브라우저 또는 기기에서 엽니다.
- Supabase 환경변수가 없는 상태에서도 앱이 localStorage 모드로 열리는지 별도로 확인합니다.

## 테스트 데이터 예시

### 세션

| 항목 | 예시 |
| --- | --- |
| 학교명 | 테스트중학교 |
| 검사명 | 2027 결핵검진 QA |
| 운영일 | 2027-06-24 |
| 장소 | 보건실 앞 검진장 |
| 담당자 | 보건교사 |
| 대상 학년 | 1, 2, 3 |

### 학급

| 순서 | 학급 | 비고 |
| --- | --- | --- |
| 1 | 1-1 | 정상 진행 |
| 2 | 1-2 | 미도착 처리 |
| 3 | 1-3 | 지연 표시 |
| 4 | 2-1 | 완료 처리 |
| 5 | 2-2 | 다음 학급 |
| 6 | 3-1 | 다다음 학급 |

### 학생 명렬표

테스트 명렬표에는 실제 학생 정보를 쓰지 않고 아래처럼 가상 이름을 사용합니다.

| 학년 | 반 | 번호 | 이름 | 테스트 상태 |
| --- | --- | --- | --- | --- |
| 1 | 1 | 1 | 테스트학생01 | 완료 |
| 1 | 1 | 2 | 테스트학생02 | 결석 |
| 1 | 1 | 3 | 테스트학생03 | 대기 |
| 1 | 2 | 1 | 테스트학생04 | 미완료 |
| 1 | 2 | 2 | 테스트학생05 | 완료 |
| 2 | 1 | 1 | 테스트학생06 | 완료 |

## 1. 검진 준비 시나리오

| 확인 항목 | 기대 결과 |
| --- | --- |
| Home에서 새 검진 시작을 누른다 | 검사 유형 선택 후 준비 화면으로 이동한다 |
| 학교 설정을 입력한다 | 학교명, 기본 장소, 기본 시간, 안내 문구가 저장된다 |
| 검사 조건을 입력한다 | 검사명, 운영일, 대상 학년, 장소가 현재 세션에 반영된다 |
| 대상 학급을 설정한다 | 학급 목록이 화면에 표시되고 가로 스크롤 없이 보인다 |
| 학급별 검진 순서를 입력한다 | 순서가 저장되고 시간표 생성에 반영된다 |
| 시간표 생성을 실행한다 | 학급별 검진 이동표가 생성된다 |
| 새로고침한다 | 선택한 세션과 입력 데이터가 유지된다 |

## 2. 검진 당일 시나리오

| 확인 항목 | 기대 결과 |
| --- | --- |
| 운영센터를 연다 | 현재 학급, 다음 학급, 진행률, 남은 학급 정보가 보인다 |
| 명렬표를 업로드한다 | 학생 목록과 상태 요약이 세션 기준으로 표시된다 |
| 현재 학급을 `1-1`로 지정한다 | 현재 학급 카드와 현황판에 `1-1`이 표시된다 |
| 현재 학급 완료를 누른다 | `1-1`이 완료 학급으로 표시되고 다음 학급이 앞으로 이동한다 |
| `1-2`를 미도착으로 표시한다 | 미도착 학급이 완료 학급과 다른 색상/상태로 구분된다 |
| 학생 1명을 완료 처리한다 | 학생 상태 요약의 완료 수가 증가한다 |
| 학생 1명을 결석 처리한다 | 결석 상태가 완료와 구분되어 표시된다 |
| 지연 시간을 입력한다 | 지연 상태가 운영센터와 관리자 현황판에 반영된다 |
| Tablet Mode를 연다 | 큰 글씨와 큰 버튼으로 완료/미도착/이전/다음 조작이 가능하다 |
| 태블릿에서 완료를 누른다 | 운영센터와 관리자 현황판에 같은 상태가 반영된다 |

## 3. 현황 공유 시나리오

| 확인 항목 | 기대 결과 |
| --- | --- |
| 교사용 현황판을 연다 | 학생 이름 없이 현재/다음 학급과 안내 문구가 보인다 |
| 관리자 현황판을 연다 | 진행률, 완료 학급, 미도착 학급, 지연 상태, 최근 로그가 보인다 |
| Display Portrait를 연다 | 보건실 세로형 화면에서 현재 학급이 가장 크게 보인다 |
| Display Landscape를 연다 | 교무실 가로형 화면에서 현재/다음/완료/남은 학급과 진행률이 보인다 |
| 운영센터에서 현재 학급을 변경한다 | 교사용/관리자/Display 화면이 같은 학급을 표시한다 |
| 운영센터에서 안내 문구를 변경한다 | 공유 화면의 안내 문구가 같은 내용으로 표시된다 |
| 공용 화면을 확인한다 | 학생 이름, 번호, 메모가 노출되지 않는다 |

## 4. 종료 후 시나리오

| 확인 항목 | 기대 결과 |
| --- | --- |
| 모든 학급을 완료 처리한다 | 진행률이 100% 또는 완료 상태에 가깝게 표시된다 |
| 운영 보고서를 연다 | 학교명, 검사명, 운영일, 출력일, 요약 카드가 보인다 |
| 학급별 결과를 확인한다 | 완료/미완료/미도착 상태가 학급 단위로 정리된다 |
| 특이사항 또는 개선사항을 입력한다 | 저장 후 새로고침해도 내용이 유지된다 |
| 인쇄 미리보기를 연다 | 버튼 등 화면용 요소가 숨겨지고 A4 기준으로 보인다 |
| PDF 저장을 선택한다 | 보고서가 학교 문서처럼 읽히는 형태로 저장된다 |
| 브라우저를 새로고침한다 | 세션, 명렬표, 운영상태, 보고서 내용이 유지된다 |

## 5. 예외 상황 체크리스트

| 예외 상황 | 확인 방법 | 기대 결과 |
| --- | --- | --- |
| 결석 학생 | 학생 상태를 결석으로 변경 | 완료 학생과 구분되고 공용 화면에는 이름이 노출되지 않는다 |
| 미도착 학급 | 학급을 미도착으로 표시 | 운영센터, 관리자 현황판, Display에서 미도착 상태를 확인할 수 있다 |
| 검사 지연 | 지연 시간을 5분 이상 입력 | 관리자 현황판과 Display Landscape에서 지연 정보가 보인다 |
| Supabase 미설정 | 환경변수 없이 앱 실행 | 앱이 깨지지 않고 localStorage 모드로 동작한다 |
| Supabase 연결 실패 | 네트워크를 끊거나 잘못된 설정으로 실행 | 화면이 멈추지 않고 fallback 안내 또는 기존 저장 데이터로 동작한다 |
| 모바일 화면 | 휴대폰 크기에서 Home, 현황판, 보고서 확인 | 가로 스크롤이나 버튼 잘림이 없다 |
| 태블릿 화면 | 세로/가로 태블릿에서 운영센터와 Tablet Mode 확인 | 핵심 버튼과 상태 카드가 한 화면에서 읽힌다 |
| 새로고침 | 운영 중 각 화면에서 새로고침 | 현재 세션과 운영 상태가 유지된다 |
| 공용 화면 개인정보 | 교사용/관리자/Display 화면 확인 | 학생 이름, 번호, 메모가 표시되지 않는다 |

## 완료 기준

- 준비, 당일 운영, 현황 공유, 종료 후 보고서 흐름이 끊기지 않는다.
- localStorage 모드에서 새로고침 후 데이터가 유지된다.
- Supabase 모드에서 여러 기기가 같은 세션 데이터를 읽고 쓴다.
- Supabase 미설정 상태에서도 앱이 깨지지 않는다.
- 모바일/태블릿에서 가로 스크롤, 버튼 잘림, 주요 정보 가림이 없다.
- 공용 화면에는 학생 개인정보가 노출되지 않는다.
- 운영 보고서는 인쇄 또는 PDF 저장 시 읽기 쉬운 문서 형태를 유지한다.
