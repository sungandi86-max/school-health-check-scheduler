# UX Guide

School Health Hub의 교사용, 보건교사용, 관리자용 화면을 설계하고 점검할 때 사용하는 UX 원칙입니다.

## 교사용 화면 UX 원칙

교사용 현황판은 수업 중 모바일로 확인하는 화면입니다.
따라서 긴 설명보다 즉시 판단할 수 있는 정보가 먼저 보여야 합니다.

정보 우선순위:

1. 지금 이동 필요한 학급
2. 현재 검사 학급
3. 다음 검사 학급
4. 미도착 학급
5. 지연 시간
6. 보건실 안내 문구

설계 기준:

- 학생 이름 전체 목록은 기본 노출하지 않습니다.
- 학급 단위 현황을 중심으로 표시합니다.
- 버튼보다 정보 표시를 우선합니다.
- 모바일 360px 이상 화면에서 핵심 카드가 잘리지 않아야 합니다.
- 미도착 학급은 색상뿐 아니라 텍스트로도 상태를 표시합니다.

## 보건교사용 운영 화면 UX 원칙

보건교사용 운영센터는 검진 당일 현장에서 빠르게 조작하는 화면입니다.
태블릿 또는 노트북에서 손으로 누를 수 있는 크기와 명확한 상태 구분이 중요합니다.

설계 기준:

- 현재 검사 지정, 완료 처리, 미도착 표시 버튼은 시각적으로 구분합니다.
- 주요 버튼은 최소 44px 이상의 터치 영역을 유지합니다.
- 학생 체크 목록은 내부 스크롤로 관리해 화면 전체가 과도하게 길어지지 않게 합니다.
- 운영 로그와 메모는 긴 문장을 읽기 쉽도록 줄간격을 넉넉히 둡니다.
- 저장 실패나 Supabase fallback 안내는 화면을 막지 않고 보조 문구로 표시합니다.

## 관리자 화면 UX 원칙

관리자/교무실 현황판은 PC 또는 교무실 모니터에서 전체 흐름을 확인하는 화면입니다.
세부 조작보다 진행률과 위험 신호를 크게 보여주는 것이 우선입니다.

정보 우선순위:

1. 전체 진행률
2. 완료 학생 / 전체 학생
3. 미검 학생
4. 완료 학급 / 전체 학급
5. 미도착 학급
6. 지연 시간
7. 최근 운영 로그

설계 기준:

- 핵심 숫자는 카드형으로 크게 표시합니다.
- 미도착, 지연, 미검 상태는 텍스트와 색상을 함께 사용합니다.
- 큰 화면에서는 카드 간 여백과 글자 크기를 늘립니다.
- 상세 목록은 핵심 숫자보다 아래에 배치합니다.

## 접근성 기준

- 모든 주요 버튼에는 동작을 알 수 있는 텍스트 또는 `aria-label`을 제공합니다.
- 입력 요소는 `label`과 연결합니다.
- 키보드 포커스가 명확히 보이도록 유지합니다.
- 색상만으로 상태를 구분하지 않고 상태 텍스트를 함께 표시합니다.
- 흐린 보조 텍스트는 충분한 대비를 유지합니다.

## 반응형 기준

- 모바일: 360px 이상에서 핵심 카드와 버튼이 잘리지 않게 합니다.
- 태블릿: 운영센터 버튼을 크게 유지하고 학급 목록과 체크 영역을 빠르게 조작할 수 있게 합니다.
- PC: 관리자 현황판의 핵심 숫자가 한눈에 들어오도록 카드 그리드를 사용합니다.
- 큰 화면: 교무실 모니터에서도 여백과 글자 크기를 충분히 키웁니다.

## Codex 작업 후 UX 점검 절차

1. 변경 화면을 모바일, 태블릿, PC 기준으로 확인합니다.
2. 핵심 정보가 화면 첫 영역에 보이는지 확인합니다.
3. 버튼을 키보드 Tab으로 이동하며 포커스가 보이는지 확인합니다.
4. 색상 없이 텍스트만 읽어도 상태를 알 수 있는지 확인합니다.
5. `npm run lint`, `npm run build`, 필요한 경우 `npm run test`를 실행합니다.
