# Workspace Guide

## 목적

Workspace는 School Health Hub 안에서
하나의 업무를 처음부터 끝까지 수행하는 독립적인 작업 공간입니다.

Workspace는 단순한 화면이 아니라

"업무 흐름"을 제공합니다.

---

# 설계 철학

모든 Workspace는

준비

↓

계획

↓

현장 운영

↓

현황 공유

↓

결과

라는 동일한 업무 흐름을 따릅니다.

사용자는 현재 어느 단계인지 항상 이해할 수 있어야 합니다.

---

# 기본 구조

모든 Workspace는 다음 구조를 권장합니다.

운영 대시보드

↓

운영 준비

↓

운영 계획

↓

현장 모드

↓

현황 공유

↓

운영 결과

---

# 운영 대시보드

Workspace의 시작 화면입니다.

역할

- 최근 작업
- 이어하기
- 새 작업 시작
- 현재 세션
- 현황판 진입

사용자는 언제든 운영 대시보드로 돌아올 수 있어야 합니다.

---

# 운영 준비

예시

- 학교 설정
- 검사 조건
- 대상 설정
- 기본 옵션

운영을 시작하기 전 필요한 정보를 준비합니다.

---

# 운영 계획

예시

- 시간표 생성
- 순서 설정
- 이동 계획

현장 운영을 위한 계획을 생성합니다.

---

# 현장 모드

현장에서 사용하는 화면입니다.

원칙

- 큰 버튼
- 큰 글씨
- 최소 입력
- 빠른 체크

복잡한 설정은 포함하지 않습니다.

---

# 현황 공유

운영 상황을 공유하는 화면입니다.

예시

- 교사용 현황판
- 관리자 현황판
- Display

학생 개인정보는 최소한으로 표시합니다.

---

# 운영 결과

운영 종료 후 확인하는 단계입니다.

예시

- 운영 보고서
- 출력
- CSV
- 통계

---

# UI 원칙

모든 Workspace는

- 같은 사이드바 구조
- 같은 단계 흐름
- 같은 버튼 위치
- 같은 색상 체계

를 유지합니다.

사용자가 새로운 Workspace를 사용하더라도
학습 없이 사용할 수 있어야 합니다.

---

# 확장 원칙

새로운 기능은
기존 Workspace 안에 무분별하게 추가하지 않습니다.

새로운 업무 흐름이라면
독립적인 Workspace를 생성합니다.

예)

별도검사 Workspace

교직원 건강검진 Workspace

보건교육 Workspace

감염병 대응 Workspace

AED 관리 Workspace

---

# 현재 Workspace

현재 구현

- 별도검사 Workspace

향후 예정

- 교직원 건강검진
- 보건교육
- 감염병 관리
- AED 관리

---

# 개발 원칙

Workspace는

"기능"

이 아니라

"업무 전체"

를 담당하는 공간입니다.

모든 기능은 Workspace의 업무 흐름 안에서
자연스럽게 연결되어야 합니다.
