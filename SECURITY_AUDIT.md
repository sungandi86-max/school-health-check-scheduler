# Security Audit Checklist

## 목적

School Health Hub 릴리즈 전 보안과 개인정보 노출 여부를 점검하기 위한 체크리스트입니다.

## 1. 개인정보 노출 점검

- [ ] 학생 이름이 공용 화면에 표시되지 않는다.
- [ ] 학번이 공용 화면에 표시되지 않는다.
- [ ] Display 화면에는 학급 단위 정보만 표시된다.
- [ ] 관리자 현황판에는 학생별 개인정보가 기본 표시되지 않는다.
- [ ] 교사용 현황판에는 필요한 최소 정보만 표시된다.

## 2. URL 점검

- [ ] URL에 학생 이름이 포함되지 않는다.
- [ ] URL에 학번이 포함되지 않는다.
- [ ] URL에 민감한 세션 정보가 직접 노출되지 않는다.

## 3. Console 점검

- [ ] console.log에 학생 이름이 출력되지 않는다.
- [ ] console.log에 학번이 출력되지 않는다.
- [ ] console.log에 Supabase 키나 환경변수가 출력되지 않는다.

## 4. 저장소 점검

- [ ] localStorage에 저장되는 항목을 확인했다.
- [ ] Supabase 사용 시 RLS 정책을 확인했다.
- [ ] service_role key가 프론트엔드에 포함되지 않는다.
- [ ] .env 파일이 Git에 포함되지 않는다.

## 5. 현장 모드 점검

- [ ] 현장 모드는 운영에 필요한 최소 학생 정보만 표시한다.
- [ ] 현장 모드에서 불필요한 개인정보가 노출되지 않는다.
- [ ] 공용 Display와 현장 모드의 정보 범위가 구분된다.

## 6. 릴리즈 전 확인

- [ ] npm.cmd run build 성공
- [ ] 주요 화면 직접 확인
- [ ] 모바일 화면 확인
- [ ] 태블릿 화면 확인
- [ ] Display 화면 확인
- [ ] 운영 보고서 출력 화면 확인

## 7. 위험 등급

문제가 발견되면 다음 등급으로 분류합니다.

- Critical: 학생 개인정보 외부 노출 가능
- High: 접근 권한 문제 또는 Supabase 정책 문제
- Medium: 공용 화면 개인정보 노출 가능성
- Low: 문구, 안내, UI 수준의 보안 개선 필요
